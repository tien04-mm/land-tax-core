# Frontend-Backend (FE-BE) API Contracts

This document contains the complete list of RESTful API contracts for the Land Tax Management System, generated from the Spring Boot backend source code. 

## General Integration Rules

### Base URL
- **Local Dev Server**: `http://localhost:8080` (CORS is configured to allow requests from `http://localhost:3000` for FE development).
- **Public Endpoints**: `/api/public/**`, `/api/auth/**`, `/api/webhook/**`, `/api/profile/sync`, `/api/land-prices/lookup`, `/api/lands/**` do not require authentication headers.
- **Protected Endpoints**: Require authorization headers in the format below.

### Headers
Every request to a protected endpoint must include the JSON Web Token (JWT) in the `Authorization` header:
```http
Authorization: Bearer <your_jwt_access_token>
Content-Type: application/json
```

## 1. Authentication & Profile Module

Endpoints for logging in via VNeID, managing JWT lifecycles, and managing user profiles.

### POST /api/auth/qr-login
* **Description**: Log in using a QR Token generated by the VNeID Simulator.
* **Access**: Public
* **Request Body** (`AuthRequest.QrLoginRequest`):
```json
{
  "qrToken": "string"
}
```
* **Response Body** (`AuthResponse`):
```json
{
  "token": "eyJhbGciOi...",
  "refreshToken": "7c48fde2...",
  "userId": 4,
  "fullName": "Nguyễn Văn A",
  "cccdNumber": "012345678901",
  "activeRole": "ROLE_CITIZEN",
  "roles": ["ROLE_CITIZEN"]
}
```

### POST /api/auth/switch-role
* **Description**: Switch the current active role for users with multiple roles.
* **Access**: Protected (Authenticated user)
* **Request Body** (`AuthRequest.SwitchRoleRequest`):
```json
{
  "targetRole": "ROLE_TAX_OFFICER"
}
```
* **Response Body** (`AuthResponse`):
```json
{
  "token": "eyJhbGciOi...",
  "refreshToken": "7c48fde2...",
  "userId": 4,
  "fullName": "Nguyễn Văn A",
  "cccdNumber": "012345678901",
  "activeRole": "ROLE_TAX_OFFICER",
  "roles": ["ROLE_CITIZEN", "ROLE_TAX_OFFICER"]
}
```

### POST /api/auth/refresh
* **Description**: Refresh an expired Access Token using the Refresh Token.
* **Access**: Public
* **Request Body**:
```json
{
  "refreshToken": "7c48fde2..."
}
```
* **Response Body** (`AuthResponse`): Same structure as `/api/auth/qr-login`.

### POST /api/auth/logout
* **Description**: Log out and revoke the current Refresh Token.
* **Access**: Protected
* **Request Body**:
```json
{
  "refreshToken": "7c48fde2..."
}
```
* **Response Body**:
```json
{
  "message": "Đăng xuất thành công"
}
```

### POST /api/profile/sync
* **Description**: Synchronize citizen profile info directly from VNeID.
* **Access**: Protected (CCCD extracted from JWT token)
* **Request Body**: None
* **Response Body**:
```json
{
  "message": "Dong bo ho so thanh cong",
  "profile": {
    "citizenId": 4,
    "cccdNumber": "012345678901",
    "fullName": "Nguyễn Văn A",
    "email": "vana@gmail.com",
    "phoneNumber": "0987654321"
  }
}
```

### GET /api/profile
* **Description**: Retrieve the current citizen's cached profile.
* **Access**: Protected
* **Request Body**: None
* **Response Body** (`CitizenLocalEntity`):
```json
{
  "citizenId": 4,
  "cccdNumber": "012345678901",
  "fullName": "Nguyễn Văn A",
  "email": "vana@gmail.com",
  "phoneNumber": "0987654321"
}
```

## 3. Tax Declarations & Records Module

Endpoints managing tax declarations, record reviews, verification, approvals, and rejections.

### POST /api/tax/declarations
* **Description**: Submit a land tax declaration.
* **Access**: Protected (`ROLE_CITIZEN`, `ROLE_ADMIN`)
* **Request Body** (`TaxDeclarationRequest`):
```json
{
  "parcelId": 1,
  "declaredArea": 150.50,
  "attachmentIds": [12, 13]
}
```
* **Response Body** (`TaxDeclarationResponse`):
```json
{
  "recordId": 10,
  "citizenId": 4,
  "parcelId": 1,
  "declaredArea": 150.50,
  "declaredUsage": "Đất ở tại đô thị",
  "status": "SUBMITTED",
  "declarationNotes": null,
  "createdAt": "2026-05-22T04:30:00"
}
```
*Note*: If `declaredArea` differs by >2% compared to actual DB area size, `status` returns `FRAUD_SUSPECTED`.

### GET /api/tax/records/verified
* **Description**: Fetch all records verified by Land Officers, awaiting Tax Officer approval.
* **Access**: Protected (`ROLE_TAX_OFFICER`)
* **Response Body** (`List<RecordEntity>`):
```json
[
  {
    "recordId": 10,
    "citizenId": 4,
    "landParcelId": 1,
    "recordCategory": "TAX_DECLARATION",
    "currentStatus": "VERIFIED",
    "submittedAt": "2026-05-22T04:30:00",
    "taxDeclaration": {
      "declarationId": 5,
      "declaredArea": 150.50,
      "declaredUsage": "Đất ở tại đô thị",
      "declarationNotes": null
    }
  }
]
```

### PUT /api/tax/records/{id}/approve
* **Description**: Approve a verified declaration, triggering invoice (tax bill) generation.
* **Access**: Protected (`ROLE_TAX_OFFICER`, `ROLE_ADMIN`)
* **Request Body** (`ReviewDeclarationRequest` - optional):
```json
{
  "processorNotes": "Giao diện kiểm duyệt hợp lệ"
}
```
* **Response Body**:
```json
{
  "recordId": 10,
  "oldStatus": "VERIFIED",
  "newStatus": "APPROVED",
  "paymentStatus": "AWAITING_PAYMENT",
  "approvedBy": "012345678901",
  "message": "Declaration #10 has been approved successfully"
}
```

### PUT /api/tax/records/{id}/verify *(Quirk: Mapped to Reject Record logic)*
* **Description**: Reject a tax declaration.
* **Access**: Protected (`ROLE_TAX_OFFICER`)
* **Request Body** (`ReviewDeclarationRequest` - REQUIRED):
```json
{
  "processorNotes": "Diện tích kê khai không chính xác"
}
```
* **Response Body**:
```json
{
  "recordId": 10,
  "oldStatus": "VERIFIED",
  "newStatus": "REJECTED",
  "reason": "Diện tích kê khai không chính xác",
  "rejectedBy": "012345678901",
  "message": "T khai m #10  b t ch i"
}
```

### GET /api/tax/declarations/my-history
* **Description**: Get declaration logs for the currently logged-in citizen.
* **Access**: Protected (`ROLE_CITIZEN`)
* **Response Body**: `List<TaxDeclarationResponse>` (Similar to POST `/api/tax/declarations` response).

### GET /api/tax/declarations/{id}
* **Description**: Get full declaration detail.
* **Access**: Protected (`ROLE_CITIZEN`)
